## 北极星（唯一硬门槛）

**玩约 3 分钟后：你还想不想再来一把？**

- **过**：想，或已经下意识点了「再来」
- **不过**：没意思 / 不想再点 → **整段设计作废**，不堆关卡、不堆 meta

次要（过北极星后再看）：

- 会不会**主动再开一局**（同一会话）
- 明天还想不想打开

## 什么叫「有意思」（C 条，新核心至少 3/4）

| # | 标准 | 闭环核心如何体现 |
|---|------|----------------|
| C1 | 即时后果 | 下雨田变绿、蒸发海面降、渗水海面升——都肉眼可见 |
| C2 | 权衡 | 水有限：先救哪块田 / 囤云还是等海回水 |
| C3 | 变化 | 正午蒸发快、海低了蒸发慢，同样操作结果不同 |
| C4 | 可读推进 | 每块田湿度条 + 绿带、白天进度、全绿保持倒计时 |

## 核心设计红线（C 版专属，别再违反）

1. **水守恒**：sea + cloud + fields 恒等于 TOTAL。任何新元素（河/雪/地下水）都只是**多一个水库 + 多一条流**，不许凭空产/灭水。见 `ELEMENTS.md`。
2. **联系可见**：三态之间的流动必须画出来（升腾水汽、下落雨、回海细流），不能只靠顶部守恒条这类抽象 UI。
3. **给自然元素留余地**：模型按「水库列表 + 流列表」扩展，加元素是**加法**不是重写。
4. **永不失败**：干田只发黄会恢复；天黑没平衡是「再试」，不是死亡。

## 明确不算成功

- 测试绿、关卡多、系统多
- 「有教育意义」但你自己不想再玩
- 文档写得很漂亮

## 你怎么评当前版本

1. `npm run dev` → 直接进闭环核心
2. 拖动画布瞄准，按住右下角 ☔ 下雨（看海面降、水汽升、雨落、回海细流）
3. 让每块田同时落进绿带、保持几秒 → 全谷开花
4. 只回答一句：**还想再来一把吗？**（想 / 不想 / 为什么）
